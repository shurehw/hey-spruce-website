NFPA 96 documentation workflow: certificates, evidence, and inspection-ready records

NFPA 96 compliance fails operationally when documentation is missing, inconsistent, or not retrievable. The fix is a workflow: schedule → perform work → capture evidence → store certificates → log exceptions → verify closure.

What ops needs to be able to prove

  • Which locations were serviced, when, and by whom
  • What scope was performed (hood/duct/fan as applicable)
  • Where the certificate lives and how to retrieve it fast
  • What exceptions occurred and how they were closed

Exception handling: missing docs are a work order

If a certificate is missing, that’s not “admin.” It’s an open exception with an owner and due date. Closure requires the document linked to the location record.

Location Exception Owner Closure evidence Status
Location B Missing NFPA certificate for last service Vendor Certificate uploaded + linked Open

This is the same pattern used in the platform and vendor management, and it shows up in the inspection readiness case study: full-service group inspection readiness.
